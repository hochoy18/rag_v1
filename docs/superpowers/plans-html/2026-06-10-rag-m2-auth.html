Goal

把 v1-scope 决策 #5（鉴权：随机 token + SHA-256 哈希 + DB 查表，不是 JWT）、#6（密码：argon2id 哈希）、#7（越权 session_id → 404，不暴露存在性）落成可执行鉴权层：

  • POST /api/auth/register — 注册用户（密码强度校验 + 用户名归一 + argon2id 哈希，返回用户基本信息）
  • POST /api/auth/login — 登录（账号锁检查 + 密码校验，签发 32 字节随机 token，SHA-256 哈希后存 DB，软吊销该 user 旧 session → 单设备策略，返回 token）
  • POST /api/auth/logout — 登出（按 Authorization header 查 session，软吊销 → 204）
  • GET /api/auth/me — 鉴权 smoke endpoint（M3+ protected endpoint 样板；返回当前 user）
  • FastAPI Depends(get_current_user) — 路由级注入，从 Authorization: Bearer *** 查会话、校验过期时间、必要时 BackgroundTasks 触发 extend_session`
  • 限速：register 5 次/分钟/IP，login 10 次/分钟/IP（slowapi）
  • 账号锁：连续 5 次密码错误 → 锁定 15 分钟（M1 字段 users.failed_login_attempts / users.locked_until）

    • 不包含（其他 M 负责）：M1 schema 表定义（已有 users / auth_sessions 表 + is_revoked / email / failed_login_attempts / locked_until / ip_address / user_agent 字段）、M3–M12 业务路由的 auth 集成（各 M 通过 Depends(get_current_user) 接入）、OAuth2 第三方登录、密码找回/重置流程、admin 角色管理、JWT 兼容层（V1 决策 #5 已否决 JWT）、Dockerfile（M8/M12 阶段补）

async def get_current_user(

    background_tasks: BackgroundTasks = BackgroundTasks(),
