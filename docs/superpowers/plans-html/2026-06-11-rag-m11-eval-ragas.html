Goal

把 v1-scope 决策 #11 落成可执行代码契约：

  • 一个 Click CLI（python -m app.eval.ragas_run），跑 golden set 算 RAGAS 三指标（faithfulness / answer_relevancy / context_precision），输出 JSON + HTML 报告
  • 一个 50 题 golden set（JSONL 格式，3 源 × 3 类问题矩阵 + 5 chitchat + edge），git 化锁版本。r2-2026-06-12 新-1 已修：fixture 重排 q001-q005 = 5 chitchat + q006-q050 = 45 rag（CI 前 20 = q001-q020 包含 5 chitchat + 15 rag；nightly = 50 题全部）
  • 一个 CI gate（CI faithfulness < 0.65 / nightly < 0.7 抛 GateViolationError、exit code 1），M12 Hardening 接 CI
  • RAGAS judge LLM 显式注入（不读 env 里的 OpenAI key），复用 M3 make_llm("judge")，强制不接 Langfuse callback（避免污染业务 trace 看板）
  • 跑 RAGAS 时把 M10 Langfuse trace_id 写入报告（用 langfuse 2.50+ 公开 API get_client().get_current_trace_id() + try/except 包装），建立"评测 → 业务 trace"双向追溯

    • 不包含（其他 M 负责）：M10 业务级 Langfuse trace 实现（只读已有 trace_id）、M12 CI workflow（只产出 gate 钩子）、M4–M6 真实 ingest 数据（评测用 50 题样本够）、M9 UI 集成（HTML 报告独立产物）。

  • app/eval/ 全部依赖 M3/M7/M10 接口，不引入新端口/新服务

def load_golden_set(path: str) -> list[GoldenItem]:
    items = []
    with open(path) as f:
        for i, line in enumerate(f, 1):
            if not line.strip():
                continue
            item = GoldenItem.model_validate_json(line)
            items.append(item)
    return items
