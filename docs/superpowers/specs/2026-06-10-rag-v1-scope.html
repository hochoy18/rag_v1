关键架构决策：

  • 三库职责清晰：LlamaIndex（ingest 阶段 Reader + Splitter）/ langchain（LLM 客户端抽象层）/ langgraph（query 阶段 workflow 编排）
  • Ingest / query 解耦：客户端上传 → /api/ingest → BackgroundTasks 入队 → 异步 worker
  • 7 节点 RAG workflow：load_memory → classify（启发式+LLM 兜底）→ rewrite(1-3 句) → retrieve(kNN top_k=10) → rerank(LLM 0-10 top-5) → answer([1][2][3] 引用) → save_memory
  • PG 不可达降级 SQLite：make_checkpointer() 自动 fallback
  • 可观测双轨：Langfuse（在线 trace）+ RAGAS（离线 golden set）

2. 模块树

app/
├── api/              FastAPI 路由 (auth, sessions, chat, ingest, health)
├── auth/             用户系统 (service, tokens, deps)
├── db/               数据层 (models, session, migrations/)
├── graph/            LangGraph 工作流
│   ├── state.py      TypedDict RagState
│   ├── nodes.py      7 节点函数
│   ├── edges.py      条件边（classify → 检索/闲聊）
│   ├── workflow.py   compile + checkpointer 注入
│   ├── checkpointer.py  Postgres/SQLite 降级
│   └── errors.py     自定义异常 + fallback_answer
├── ingest/           数据源 + 管线 (file, url, confluence + auth, pipeline, splitter)
├── retrieval/        OpenSearch 封装 (client, index, store, retriever)
├── memory/           Postgres checkpointer + thread 解析
├── llm/              LLM 工厂 (factory, prompts)
├── embedding/        TEI 客户端
├── observability/    Langfuse callback 工厂
├── eval/             RAGAS CLI 入口
└── ui/               Gradio 4 tab

隔离原则：每个模块一个 __init__.py 暴露最少公共 API；跨模块仅通过显式接口（函数签名 / Pydantic schema）；LangGraph state 用 TypedDict 集中定义，节点不私自扩展。

3. 数据流

3.1 Ingest

POST /api/ingest {source, payload, auth?}
  → 鉴权 → INSERT ingest_jobs (status=pending) → BackgroundTask
      ├─ file:    LlamaIndex SimpleDirectoryReader
      ├─ url:     httpx + auth → TrafilaturaWebReader
      └─ confluence: v2 REST → BFS 子页 → attachments
  → 元素分类 (NarrativeText/Table/Image)
  → 图片元素 → artifacts/ + image_ref 元数据（V1 不做内容问答）
  → SentenceSplitter (chunk=512, overlap=64)
  → TEI /embed (bge-m3, batch=32)
  → OpenSearch upsert {id, text, vector[1024], metadata}
  → UPDATE ingest_jobs (status=indexed|failed)

3.2 Auth

POST /api/auth/register → argon2id hash → INSERT users + auth_sessions
POST /api/auth/login    → verify password → INSERT auth_sessions
                          → token 7d 滑动 / 30d 硬上限
POST /api/auth/logout   → DELETE auth_sessions WHERE token_hash=?
GET  /api/sessions      → SELECT chat_sessions ORDER BY updated_at DESC
GET  /api/sessions/{id} → PostgresCheckpointer.get_state(thread_id)
                          → 越权 404（不暴露存在性）

3.3 Query

POST /api/chat {query, session_id?}
  → 鉴权 → resolve thread_id → LangGraph invoke
    1. load_memory   → get_state(thread_id) → 历史消息
    2. classify      → 启发式("刚才/上面/它") → chitchat
                      → LLM 兜底 → retrieve/chitchat
    3a. chitchat     → answer 直出
    3b. retrieve     → rewrite(1-3) → kNN top_k=10 → dedupe
                     → rerank(LLM 0-10 top-5) → answer([1][2][3])
    4. save_memory   → 自动 persist
  → return {answer, sources, session_id}

5. 错误处理矩阵

失败点策略
OpenSearch 不可达503 + LLM 兜底（无源引用"未检索"）
TEI embedding 超时指数退避 3 次 → 标 failed
LLM 4xx/5xxLangfuse trace → 前端 502
检索 top-k 全 0"未找到相关文档"，不编造
Postgres 不可达SQLite 降级（checkpointer）
Confluence 401/403跳过 page，记日志
Confluence 429token bucket + 退避 3 次
文档解析失败单文件不阻塞批次
附件 >50MB跳过，不进 RAG
session_id 越权404，不暴露存在性

6. 测试策略

  • 模型：MiniMax-M3（langchain ChatAnthropic）
  • 单元：pytest + pytest-asyncio（graph 节点、retrieval、auth、checkpointer roundtrip）
  • 集成：docker compose up → 真 OpenSearch/TEI/PG/Langfuse → 端到端
  • 断言：answer 含 [1][2][3] 源标记 / retrieve chunk ≥ 0 / chitchat 不调 retrieve
  • RAGAS：golden set 20-50 题（3 源 × 3 类问题），CI 门禁 faithfulness ≥ 0.7
  • TDD：superpowers 红绿循环
  • 不测：LLM 输出文本本身、Confluence API（mock）
