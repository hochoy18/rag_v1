### 关键架构决策

1. **三库职责清晰**：LlamaIndex（ingest 阶段 Reader + Splitter）/ langchain（LLM 客户端抽象层）/ langgraph（query 阶段 workflow 编排）
2. **Ingest / query 解耦**：客户端上传 → `/api/ingest` → BackgroundTasks 入队 → 异步 worker
3. **7 节点 RAG workflow**：load_memory → classify（启发式+LLM 兜底）→ rewrite(1-3 句) → retrieve(kNN top_k=10) → rerank(LLM 0-10 top-5) → answer([1][2][3] 引用) → save_memory
4. **PG 不可达降级 SQLite**：`make_checkpointer()` 自动 fallback
5. **可观测双轨**：Langfuse（在线 trace）+ RAGAS（离线 golden set）

### 隔离原则

- 每个模块一个 `__init__.py` 暴露**最少**公共 API
- 跨模块仅通过显式接口（函数签名 / Pydantic schema）
- LangGraph state 用 TypedDict 集中定义，节点不私自扩展

## 3. 数据流

### 3.1 Ingest

```
POST /api/ingest
  { source: "file"|"url"|"confluence",
    payload: {...},
    auth?: {...} }
  → 鉴权 (current_user)
  → INSERT ingest_jobs (status=pending, source, payload_hash)
  → BackgroundTask
      ├─ file:    LlamaIndex SimpleDirectoryReader
      ├─ url:     httpx + auth → TrafilaturaWebReader
      └─ confluence: v2 REST → fetch root page → BFS 子页 → fetch attachments
  → 解析（Unstructured.io / Trafilatura / Confluence storage format）
  → 元素分类（NarrativeText / Table / Image / ...）
  → 图片元素抽到 artifacts/{doc_id}/images/，元数据 image_ref
  → SentenceSplitter (chunk=512, overlap=64)
  → TEI /embed (bge-m3, batch=32)
  → OpenSearch PUT /chunks/_doc
       { id, text, vector[1024], metadata: {source, doc_id, page_id, image_ref, ...} }
  → UPDATE ingest_jobs (status=indexed|failed, chunks_count, error?)
```

### 3.2 Auth

```
POST /api/auth/register {username, password}
  → 校验（用户名唯一、密码强度）
  → argon2id hash
  → INSERT users
  → INSERT auth_sessions (token=secrets.token_urlsafe(32), token_hash=sha256(token))
  → return {user_id, token}

POST /api/auth/login {username, password}
  → 查 users WHERE username
  → argon2id verify
  → UPDATE users.last_login_at
  → INSERT auth_sessions
  → return {user_id, token}

POST /api/auth/logout
  → DELETE auth_sessions WHERE token_hash=?

GET /api/sessions
  → 鉴权
  → SELECT * FROM chat_sessions
    WHERE user_id=current_user
    ORDER BY updated_at DESC
  → return [{session_id, title, updated_at, thread_id}]

GET /api/sessions/{id}
  → 鉴权 + 归属校验
  → PostgresCheckpointer.get_state(thread_id)
  → return {session_id, thread_id, messages[]}
```

### 3.3 Query（含 memory + 持久化）

```
POST /api/chat
  { query: "去年 Q3 vs Q4 各业务线对比", session_id?: "..." }
  → 鉴权
  → session_id 缺 → INSERT chat_sessions (新 thread_id=uuid4) → 返回
  → session_id 有 → 查 chat_sessions.thread_id (校验归属 current_user)
  → LangGraph invoke({query, user_id, session_id},
                    config={"configurable": {"thread_id": thread_id}})
       ┌────────────────────────────────────────────┐
       │ 1. load_memory                            │
       │    PostgresCheckpointer.get_state → msgs   │
       │ 2. classify                                │
       │    启发式: "刚才/上面/它" → 闲聊              │
       │    否则: llm.classify(query, history)        │
       │    intent ∈ {retrieve, chitchat}           │
       │ 3a. (chitchat)                             │
       │     answer = llm.chat(msgs + query)        │
       │ 3b. (retrieve)                             │
       │     rewrites = llm.rewrite(query) [1..3]   │
       │     chunks  = []                            │
       │     for r in rewrites:                      │
       │       chunks += retrieve(r, top_k=10)       │
       │     chunks = dedupe(chunks)                 │
       │     chunks = llm.rerank(query, chunks) top5 │
       │     answer  = llm.answer(query, chunks)     │
       │ 4. save_memory                              │
       │    checkpointer 自动 persist                │
       │ 5. Langfuse trace (整条链)                  │
       └────────────────────────────────────────────┘
  → return {answer, sources: [{chunk_id, doc_id, image_ref?, score}], session_id}
```

## 5. 错误处理矩阵

| 失败点 | 策略 |
|--------|------|
| OpenSearch 不可达 | 503 + 管理员提示；query 走纯 LLM 兜底（无源引用，明示"未检索"） |
| TEI embedding 超时 | 指数退避 3 次 → 报错；ingest 任务标 failed，可重试 |
| LLM 4xx/5xx | Langfuse 记 trace；前端 502；不静默 |
| 检索 top-k 全 0 分 | answer 显式说"未找到相关文档"，不编造 |
| Postgres 不可达 | checkpointer 不可用 → 降级本地 SQLite；UI 告警"持久化降级" |
| Confluence API 401/403 | 跳过该 page，记日志，最终任务报告"无权限 X 个" |
| Confluence 429 | token bucket + 指数退避，重试 3 次后跳过 |
| 文档解析失败（PDF 损坏） | 单文件失败不阻塞批次；记日志 + 状态 |
| 附件超大 (>50MB) | 跳过，记日志，不进 RAG |
| 用户未登录访问 /api/chat | 401，前端跳登录 |
| session_id 越权（不属于 current_user） | 404，**不暴露**存在性 |

## 6. 测试策略

- **LLM 模型**：minimax-cn `MiniMax-M3`（langchain `ChatAnthropic`）
- **单元（pytest + pytest-asyncio）**：每模块单测
  - 重点：graph 节点、retrieval 包装、auth 令牌、checkpointer roundtrip
- **集成（docker compose up）**：起真实 OpenSearch + TEI + Postgres + Langfuse，端到端
  - ingest: file/url/confluence 各跑一份 sample
  - chat: 单轮 / 多轮 thread 复用 / 闲聊路径
  - 鉴权: 401/404/过期
- **断言结构而非内容**
  - answer 字符串含 `[1]` `[2]` `[3]` 源标记
  - retrieve 返回 chunk 数 ≥ 0
  - 闲聊路径不调 retrieve
- **RAGAS 评测**
  - golden set：20-50 题（覆盖 3 种源、3 类问题）
  - 指标：faithfulness、answer_relevancy、context_precision
  - CI 门禁：faithfulness ≥ 0.7
- **TDD 红绿循环** —— 沿用 superpowers 流程
- **不测的内容**：LLM 输出文本本身（不可控）、Confluence API 响应（mock）
